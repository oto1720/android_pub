# 要件: ポータルから記事詳細への遷移を廃止

## 概要

ポータル画面では記事タップで詳細（Digest）画面へ遷移しない。タイトル・プレビューを見て「スロットに追加」した記事のみ、積読画面から読めるようにする。

---

## 背景・課題

- 従来: ポータルで記事タップ → 全文読める → 積読に追加するか選べる
- **問題**: 全文が読めるため積読スロットの意味が薄れ、「読んだつもり」を防ぐという課題解決が弱い

---

## 変更後の設計

### ポータル画面

| 項目 | 内容 |
|------|------|
| **表示** | タイトル・冒頭文（プレビュー）のみ。全文は表示しない |
| **記事タップ** | 何もしない（Digest へ遷移しない） |
| **スロットに追加** | 気になった記事を積読スロットに追加。追加後は積読画面で読む |

### 積読画面

| 項目 | 内容 |
|------|------|
| **記事タップ** | Digest 画面へ遷移し、本文を読んで消化フローへ進む（従来どおり） |

---

## 期待される効果

1. **情報の制限**: 全文はスロットに入れた記事だけ。ポータルでは「選ぶ」だけ
2. **消化の強制**: 読む＝スロット経由＝消化フローに入る
3. **積読スロットの意味**: 「読む権利」を獲得する場所として明確化

---

## 備考

- 冒頭文（プレビュー）の実装は、Qiita API の `body` 取得を追加する場合に検討
- 現状はタイトル・タグのみで「選ぶ」判断が可能
