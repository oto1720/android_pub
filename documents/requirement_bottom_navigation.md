# 要件: ボトムナビゲーション

## 概要

アプリの主要3画面（Portal / 積読 / 血肉）を、画面下部の固定ナビゲーションバーで切り替えられるようにする。設計ドキュメント（design.md）で想定されている「積読リストアイコン」「血肉リストアイコン」による遷移を、ボトムナビとして実装する。

---

## 背景・現状

- 現在、ボトムナビゲーションは未実装
- MainActivity の Scaffold には `bottomBar` が渡されていない
- Portal の `onNavigateToTsundoku` / `onNavigateToDone` は AppNavHost から渡されているが、PortalScreenContent では未使用（将来実装予定として PR レビューで言及）
- 画面間の遷移は各画面内のボタン・リンクに依存している

---

## 機能要件

### FR-1: タブ構成

| 項目 | 内容 |
|------|------|
| **タブ数** | 3タブ |
| **タブ1** | **Portal**（ポータル / トレンド一覧） |
| **タブ2** | **積読**（積読リスト、最大5枠） |
| **タブ3** | **血肉**（消化済み一覧） |
| **ルート** | それぞれ `Screen.Portal` / `Screen.Tsundoku` / `Screen.Done` に対応 |

### FR-2: 表示・非表示

| 項目 | 内容 |
|------|------|
| **表示する画面** | Portal / Tsundoku / Done のいずれかが表示されているとき |
| **非表示にする画面** | Digest（記事詳細・消化） / DoneDetail（消化済み詳細） が表示されているとき |
| **理由** | 詳細画面はスタック上にプッシュされるため、ボトムナビは隠し、戻る操作を優先する |

### FR-3: タップ時の挙動

| 項目 | 内容 |
|------|------|
| **同一タブタップ** | 何もしない（またはトップへスクロール等、将来拡張可） |
| **他タブタップ** | 該当画面へ遷移し、バックスタックを適切にクリアする |
| **詳細画面表示中** | 他タブタップ時は、詳細画面を pop してから該当タブへ遷移 |

### FR-4: 選択状態の表示

| 項目 | 内容 |
|------|------|
| **現在表示中のタブ** | 視覚的に選択状態（例: アイコン・ラベルの色変更、インジケータ）で示す |
| **未選択タブ** | 選択状態と区別できるデザイン |

### FR-5: 起動時の初期タブ

| 項目 | 内容 |
|------|------|
| **初期タブ** | Portal（既存の `startDestination = Screen.Portal.route` を維持） |

## 非機能要件

### NFR-1: バックスタックの扱い

| シナリオ | 期待動作 |
|----------|----------|
| Portal 表示中に積読タブをタップ | Tsundoku 画面へ遷移。Portal はバックスタックに残すか、タブ切り替えとしてクリアする（実装方針で決定） |
| Digest 表示中に積読タブをタップ | Digest を pop し、Tsundoku を表示 |
| Digest 表示中にバックボタン | 既存どおり Portal（または遷移元）へ戻る |

### NFR-2: デザイン

| 項目 | 内容 |
|------|------|
| **Material Design** | Material3 の `NavigationBar` コンポーネントを使用することを推奨 |
| **テーマ** | `MaterialTheme.colorScheme` に準拠 |
| **アクセシビリティ** | 各タブに `contentDescription` を設定 |

### NFR-3: 既存遷移との整合

| 項目 | 内容 |
|------|------|
| **画面内の遷移** | 積読の記事タップ→Digest、積読追加→Tsundoku 等の既存コールバックは維持（※ポータルからは記事タップでDigestへ遷移しない） |
| **ボトムナビ** | それらに加え、いつでも3画面間を直接切り替え可能にする |

## 備考

### 実装方針の候補

- **A) NavController と連携**: `currentBackStackEntry` から現在ルートを判定し、表示/非表示と選択状態を制御
- **B) バックスタック**: タブ切り替え時に `popBackStack` で詳細画面をクリアし、`navigate(route)` で該当タブへ遷移。`launchSingleTop` や `popUpTo` でスタックを整理

### 参考

- design.md の画面フロー: Portal →|積読リストアイコン| Tsundoku、Portal →|血肉リストアイコン| Done
- 既存の `onNavigateToTsundoku` / `onNavigateToDone` は、ボトムナビ実装後に画面内の代替遷移として残すか、ボトムナビに統一するかは設計判断
