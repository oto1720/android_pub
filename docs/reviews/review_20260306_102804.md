## 🟠 Major Issues

### [M-1] バックグラウンド生成が ViewModel 破棄時にキャンセルされる

**ファイル**: `PortalViewModel.kt:123`
**問題**: `launch { generateQuestionUseCase(articleId) }` は `viewModelScope.launch { }` の子コルーチンのため、ユーザーがスロット追加直後に画面を離れると ViewModel が破棄され、API 呼び出し中でもキャンセルされる。

「失敗しても読了宣言時に再生成される」という設計でカバーされているため機能上は問題ないが、最適化の恩恵が受けられないケースがある。コメントにその旨を明示することを推奨。

```kotlin
// ✅ コメント追記案
// スロットに入れたら削除できないため、必ず読むことが確定する。
// 読了宣言時の待ち時間をなくすため、バックグラウンドで問いを事前生成する。
// 失敗しても読了宣言時に再生成されるため、エラーは無視する。
// ViewModel 破棄（画面離脱）時はキャンセルされるが、離脱後は恩恵がないため許容する。
launch { generateQuestionUseCase(articleId) }
```

### [M-3] `ErrorContent` / `LoadingState` 時にオフラインアイコンが表示されない
