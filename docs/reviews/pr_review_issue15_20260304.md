## 🔍 詳細レビュー

### ui/portal/PortalScreen.kt — 🟢 問題なし

#### 変更の意図
スロット満杯時の UI ロックを実装。バナー表示・カードグレーアウト・ボタン無効化の3つの視覚的フィードバックを提供。

#### 良い点

- **`isSlotFull` をループ外の Success スコープで1回だけ計算するよう修正** ✅

  以前は `items { }` ブロック内で `val isSlotFull = tsundokuCount >= MAX_TSUNDOKU_SLOTS` を各アイテム毎に計算していた（無駄な再計算）。今回 Success スコープに移動したことで N 回の計算が 1 回になった。

- **`SlotFullBanner` が `errorContainer` / `onErrorContainer` カラーを使用** ✅

  Material 3 のセマンティックカラーを使用しており、ダーク/ライトテーマ両対応が自動で保証される。

- **`ArticleCard` の `containerColor` を `surfaceVariant` に切り替え** ✅

  既存の画像オーバーレイ + ボタン無効化に加えて、カード全体の背景色も変更することでグレーアウトが視覚的に明確になった。

- **タイトルテキストの alpha = 0.4f でさらに dimming** ✅

  カードの背景変化と組み合わせることで、ユーザーが「この記事は現在追加不可」と一目で判断できる。

```kotlin
// ✅ SlotFullBanner: M3 error カラーで一貫性あり
Row(
    modifier = modifier
        .fillMaxWidth()
        .background(MaterialTheme.colorScheme.errorContainer)
        .padding(horizontal = 16.dp, vertical = 12.dp),
) {
    Icon(Icons.Default.Lock, tint = MaterialTheme.colorScheme.onErrorContainer, ...)
    Column {
        Text("積読スロットが満杯です", color = MaterialTheme.colorScheme.onErrorContainer)
        Text("スロット一覧から...", color = MaterialTheme.colorScheme.onErrorContainer.copy(alpha = 0.8f))
    }
}
```
