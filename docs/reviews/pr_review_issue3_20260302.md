### MainActivity.kt

#### 変更の意図
Hilt が DI グラフを構築できるよう `@AndroidEntryPoint` を付与。

#### 指摘事項

**[🟡 Mi-2] import 順序が Android Kotlin スタイルガイドと不一致**

```kotlin
// ❌ 現状
import android.os.Bundle
import androidx.activity.ComponentActivity
import dagger.hilt.android.AndroidEntryPoint  // ← ここに挿入された
import androidx.activity.compose.setContent
```

```kotlin
// ✅ 推奨（グループ順: android → androidx → dagger の順、各グループ内はアルファベット順）
import android.os.Bundle
import androidx.activity.ComponentActivity
import androidx.activity.compose.setContent
import androidx.activity.enableEdgeToEdge
// ...
import dagger.hilt.android.AndroidEntryPoint
```

機能上の問題はないが、コードフォーマッタ（`ktlint` / Android Studio の "Optimize Imports"）で自動整列することを推奨。

### di/AppModule.kt

#### 変更の意図
`SingletonComponent` にインストールされた基本モジュールの骨格を定義。

#### 指摘事項

**[🟡 Mi-3] ボディなしの空 object**

```kotlin
// ❌ 現状
@Module
@InstallIn(SingletonComponent::class)
object AppModule  // ボディがない
```

```kotlin
// ✅ 次のバインディング追加時の形
@Module
@InstallIn(SingletonComponent::class)
object AppModule {
    @Provides
    @Singleton
    fun provideSomething(): SomeClass = SomeClass()
}
```

現時点のセットアップ issue 範囲では許容できるが、空の `object` はコンパイル後に何もコードを生成しないため、最初のプロバイダーが追加されるまで実質的なセットアップ効果はない。次 issue で必ずバインディングを追加してください。
