### AiRepositoryImpl.kt

#### 変更の意図
`GeminiApiService.generateContent()` を呼び出し、`runCatching` でエラーを `Result.failure` に変換する。`firstText()` が `null` の場合は `IllegalStateException` を投げることで空レスポンスもエラー扱いにする。

```kotlin
override suspend fun generateQuestion(articleTitle: String): Result<String> = runCatching {
    val prompt = """
        以下の技術記事を読んだ読者に対して、理解度を確認する質問を日本語で1つ作成してください。
        ...
        記事タイトル: $articleTitle
    """.trimIndent()

    val response = geminiApiService.generateContent(
        model = MODEL_NAME,
        request = buildRequest(prompt),
    )
    response.firstText() ?: throw IllegalStateException("AIレスポンスが空です")
}
```

#### 良い点
- `runCatching` 1か所で IOException（タイムアウト）・HttpException（429 レートリミット）・IllegalStateException（空レスポンス）をすべて捕捉できる設計
- `MODEL_NAME` を `companion object` 定数に集約しており、モデル変更が1か所で済む
- `buildRequest()` ヘルパーで DTO 構築を集約し、両メソッドの重複を排除
