## 🟠 Major Issues

### [M-1] WhileSubscribed 購読停止テストの本体がコメントアウトされている

**ファイル**: `PortalViewModelTest.kt:218`

```kotlin
@Test
fun `tsundokuCount - 最後の購読者がいなくなって5秒後に購読を停止する`() = runTest(dispatchTimeoutMs = 6000L) {
    // ...
    job.cancel()
    // advanceTime(5000) // 5000ms進める  ← コメントアウト
    // runCurrent()                        ← コメントアウト
    // assertEquals(0, tsundokuCountFlow.subscriptionCount.value)  ← コメントアウト
    // NOTE: advanceTime is not working as expected...
    // We will trust the WhileSubscribed mechanism and test the subscription start.
}
```

**問題**: テストの主目的である「5秒後に購読が停止する」の検証が全てコメントアウトされており、このテストは何も検証していない（テスト名と内容が乖離している）。

**修正案**: `advanceTimeBy` を使って正しく動作させる。

```kotlin
@Test
fun `tsundokuCount - 最後の購読者がいなくなって5秒後に購読を停止する`() = runTest {
    createViewModel()
    val job = launch { viewModel.tsundokuCount.collect() }
    runCurrent()
    assertEquals(1, tsundokuCountFlow.subscriptionCount.value)

    job.cancel()
    advanceTimeBy(5_001L) // WhileSubscribed(5000) のタイムアウトを超える
    runCurrent()

    assertEquals(0, tsundokuCountFlow.subscriptionCount.value)
}
```

> ※ `dispatchTimeoutMs = 6000L` は `runTest` の別パラメータなので、`advanceTimeBy` で仮想時間を進めれば実時間を消費せず動作するはず。
