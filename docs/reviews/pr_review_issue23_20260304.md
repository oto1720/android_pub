### GenerateSummaryUseCase.kt

#### 変更の意図
記事の要約をキャッシュ優先で返す UseCase。`aiSummary != null` ならAPIをスキップし、`null` の場合のみ生成・保存を行う。

```kotlin
suspend operator fun invoke(articleId: String): Result<String> {
    val article = articleRepository.getArticleById(articleId)
        ?: return Result.failure(IllegalArgumentException("記事が見つかりません: $articleId"))

    article.aiSummary?.let { return Result.success(it) }

    return aiRepository.generateSummary(article.title).onSuccess { summary ->
        articleRepository.saveSummary(articleId, summary)
    }
}
```

#### 良い点
- `article.aiSummary?.let { return Result.success(it) }` で早期リターンによりキャッシュヒット時の分岐が明快
- `.onSuccess { saveSummary(...) }` パターンにより、APIエラー時には `saveSummary` が呼ばれないことが自動保証される
- 記事が存在しない場合も `Result.failure` に統一されており、呼び出し元で `when(result)` の網羅が容易
