### DigestRepositoryImpl.kt

#### 変更の意図
`saveQuestion` を実装。既存の `DigestEntity` があれば `copy(aiQuestion = question)` + `update`、なければ `savedAt` を現在時刻として `insert` する。

```kotlin
override suspend fun saveQuestion(articleId: String, question: String) {
    val existing = digestDao.getByArticleId(articleId)
    if (existing != null) {
        digestDao.update(existing.copy(aiQuestion = question))
    } else {
        val savedAt = SimpleDateFormat(ISO_8601_FORMAT, Locale.US)
            .apply { timeZone = TimeZone.getTimeZone("UTC") }
            .format(Date())
        digestDao.insert(DigestEntity(articleId = articleId, aiQuestion = question, savedAt = savedAt))
    }
}
```

#### 良い点
- 既存行がある場合は `copy` で他フィールド（`userMemo` / `aiFeedback`）を保持したまま `aiQuestion` のみ更新している
- `ISO_8601_FORMAT` を `companion object` 定数にまとめており、`DigestArticleUseCase` と同じフォーマット規約に準拠
- 既存の `DigestDao.update` / `DigestDao.insert` を再利用しており DAO の変更不要

#### 指摘事項

### GenerateQuestionUseCase.kt

#### 変更の意図
`DigestEntity.aiQuestion` キャッシュを優先し、未保存時のみ Gemini API で問いを生成して保存する UseCase。`GenerateSummaryUseCase` と同じ構造。

```kotlin
suspend operator fun invoke(articleId: String): Result<String> {
    val article = articleRepository.getArticleById(articleId)
        ?: return Result.failure(IllegalArgumentException("記事が見つかりません: $articleId"))

    digestRepository.getByArticleId(articleId)?.aiQuestion?.let {
        return Result.success(it)
    }

    return aiRepository.generateQuestion(article.title).onSuccess { question ->
        digestRepository.saveQuestion(articleId, question)
    }
}
```

#### 良い点
- `?.aiQuestion?.let { return }` でキャッシュチェックを1行で簡潔に表現
- `.onSuccess { }` によりAPIエラー時の `saveQuestion` 呼び出しを自動防止
- `GenerateSummaryUseCase` と同一パターンで可読性・一貫性が高い
