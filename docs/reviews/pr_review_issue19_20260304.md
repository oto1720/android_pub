## 🔍 詳細レビュー

### DigestUiState.kt

#### 変更の意図
`Loading`/`Success` の2状態から、読了フローの全ステップをモデル化した5状態に拡張。`MarkedAsDone` → `NavigateToDone` にリネームして命名をナビゲーションイベントの意図に合わせた。

#### 良い点
- 状態遷移の方向（WebLoading→ReadMode→QuestionMode→FeedbackMode）がコードのコメントと型から直感的に読める
- `FeedbackMode` に `question`・`memo`・`feedback` をすべて保持することで、UI側で状態のコピーなしにデータ参照できる

### DigestViewModel.kt

#### 変更の意図
`markAsDone()` + `ArticleRepository` 直接依存を削除し、`showQuestion()`・`submitMemo()`・`consumeArticle()` の3メソッドに分割。状態遷移は `_uiState.update {}` の型チェックガードで安全に行う。

#### 良い点

- `PLACEHOLDER_*` 定数が `companion object` に分離されており、今後のAI連携実装時に置き換えやすい
