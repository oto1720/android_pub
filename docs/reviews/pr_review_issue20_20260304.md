## 🔍 詳細レビュー

### GetDoneArticlesUseCase.kt

#### 変更の意図
`ArticleRepository.observeDoneArticles()` と `DigestRepository.observeAll()` を `combine` で JOIN し、`Flow<List<DoneItem>>` を返す。

#### 良い点
- `combine` を使った宣言的な JOIN で、どちらかのFlowが更新されると自動的に再計算される
- `mapNotNull` で DigestEntity が存在しない記事を安全に除外している

### DoneDetailViewModel.kt

#### 変更の意図

### DoneScreen.kt

#### 変更の意図
`DoneViewModel.uiState` を購読して Loading / 空状態 / 一覧の 3 パターンを表示する。空状態はガイドメッセージ、一覧は `LazyColumn` で `DoneArticleCard` を表示。

#### 良い点
- 空状態専用の UI（「まだ消化済みの記事がありません」＋ヒントテキスト）でユーザーへの案内が明確
- `LazyColumn` の `items` に `key = { it.articleId }` を指定しており、リスト更新時の再コンポーズが最適化されている
- タグ表示・日付フォーマット・メモプレビューが `DigestScreen` と一貫したデザイン
