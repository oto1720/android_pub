**重要度**: Critical

**変更箇所**: `domain/usecase/GetDoneArticlesUseCase.kt:18`

### [I-2] GenerateFeedbackUseCase と GenerateQuestionUseCase の article not found ハンドリング不統一 — 対称原理・安全原理

**重要度**: Critical

**変更箇所**: `domain/usecase/GenerateFeedbackUseCase.kt:30-31`

**比較対象（既存の類似実装）**:
- `domain/usecase/GenerateQuestionUseCase.kt:23-24`
- `domain/usecase/GenerateSummaryUseCase.kt:21-22`

**変更コード（GenerateFeedbackUseCase）**:
```kotlin
val article = articleRepository.getArticleById(articleId)
val content = article?.body ?: article?.title ?: ""   // 記事がなくても空文字でAPIを呼ぶ
return aiRepository.generateFeedback(content, question, userMemo)
```

**既存の標準パターン（GenerateQuestionUseCase / GenerateSummaryUseCase）**:
```kotlin
val article = articleRepository.getArticleById(articleId)
    ?: return Result.failure(IllegalArgumentException("記事が見つかりません: $articleId"))
// 記事がない場合は早期リターン
```

**逸脱の内容**: 3つの AI 生成 UseCase のうち2つは記事が見つからない場合に `Result.failure` を返す。`GenerateFeedbackUseCase` だけが記事なしでも空文字列 `""` を使って API を呼び出す。意味のないプロンプトで Gemini API を消費するリスクがあり、また呼び出し元の DigestViewModel でエラーが握り潰される可能性がある。

**推奨**:
```kotlin
val article = articleRepository.getArticleById(articleId)
    ?: return Result.failure(IllegalArgumentException("記事が見つかりません: $articleId"))
val content = article.body ?: article.title
```

### [I-6] DigestViewModel.isOnline が SharingStarted.Eagerly を使用 — 同型原理

**重要度**: Major

**変更箇所**: `ui/digest/DigestViewModel.kt:52-56`

**比較対象**:
- `ui/portal/PortalViewModel.kt:69-73`

**変更コード（DigestViewModel）**:
```kotlin
val isOnline: StateFlow<Boolean> = networkMonitor.isOnline
    .stateIn(
        scope = viewModelScope,
        started = SharingStarted.Eagerly,   // 他と異なる
        initialValue = true,
    )
```

**既存の標準パターン（PortalViewModel）**:
```kotlin
val isOnline: StateFlow<Boolean> = networkMonitor.isOnline
    .stateIn(
        scope = viewModelScope,
        started = SharingStarted.WhileSubscribed(5_000),  // 全 StateFlow で統一
        initialValue = true,
    )
```

**逸脱の内容**: `PortalViewModel` 内の全 StateFlow（isOnline, tsundokuCount, availableTags）は `SharingStarted.WhileSubscribed(5_000)` で統一されている。`DigestViewModel.isOnline` だけが `Eagerly` を使っており、コメントもない。`Eagerly` は ViewModel が生きている間ずっとFlowを収集し続けるため、メモリ効率が低い。

**推奨**: `SharingStarted.WhileSubscribed(5_000)` に変更する。ただし `init` ブロック内で即座にオフラインを検知する必要がある場合は、その理由をコメントで明記する。
