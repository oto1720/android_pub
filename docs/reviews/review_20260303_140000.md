### [M-1] AddToTsundokuUseCase の `currentStatus` パラメータの意図がコードから読み取れない

**ファイル**: `domain/usecase/AddToTsundokuUseCase.kt:24`

```kotlin
// ❌ 現在のコード
return repository.updateStatusIfSlotAvailable(
    id = articleId,
    newStatus = ArticleStatus.TSUNDOKU,
    currentStatus = ArticleStatus.TSUNDOKU,  // ← 何のカウントをしているのか不明
    maxSlots = MAX_TSUNDOKU_SLOTS,
)
```

`currentStatus` は「スロット数を数えるためのカウント対象ステータス」であり、「更新前の記事ステータス」ではありません。`newStatus` と `currentStatus` が同じ `TSUNDOKU` であることが混乱を招き、将来の修正者が誤って `ArticleStatus.PORTAL` などに変えてしまうリスクがあります。また、DAO の SQL に `AND articles.status = 'PORTAL'` のような制約がないため、DONE 状態の記事に対しても呼び出すと TSUNDOKU に書き換わります（現在は UI フローで防がれているが、UseCase だけでは保証されない）。

```kotlin
// ✅ 修正案: パラメータの意図をドキュメントで明示 + 元ステータスのガードを追加
/**
 * @param articleId 追加する記事のID。PORTAL状態の記事のみを対象とする。
 * @return 追加成功 true、スロット満杯で追加不可 false
 */
suspend operator fun invoke(articleId: String): Boolean {
    return repository.updateStatusIfSlotAvailable(
        id = articleId,
        newStatus = ArticleStatus.TSUNDOKU,
        currentStatus = ArticleStatus.TSUNDOKU, // スロット数カウントに使うステータス
        maxSlots = MAX_TSUNDOKU_SLOTS,
    )
}
```
