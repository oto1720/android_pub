    isMinifyEnabled = false

> `buildConfigField` を `defaultConfig` に置くことで重複を排除でき、将来 staging ビルドタイプを追加した際も漏れなく設定される。

### di/NetworkModule.kt

#### 指摘事項

**[🟡 M-3] `OkHttpClient` にタイムアウト設定がない** (`NetworkModule.kt:31-33`)

```kotlin
// ❌ 現状: デフォルト10秒タイムアウトのまま
fun provideOkHttpClient(loggingInterceptor: HttpLoggingInterceptor): OkHttpClient =
    OkHttpClient.Builder()
        .addInterceptor(loggingInterceptor)
        .build()
```

```kotlin
// ✅ 推奨: タイムアウトを明示設定
import java.util.concurrent.TimeUnit

fun provideOkHttpClient(loggingInterceptor: HttpLoggingInterceptor): OkHttpClient =
    OkHttpClient.Builder()
        .connectTimeout(30, TimeUnit.SECONDS)
        .readTimeout(30, TimeUnit.SECONDS)
        .writeTimeout(30, TimeUnit.SECONDS)
        .addInterceptor(loggingInterceptor)
        .build()
```

> デフォルトの 10s は多くのモバイルネットワーク環境で短すぎる場合がある。明示することで意図が伝わりやすくなる。
