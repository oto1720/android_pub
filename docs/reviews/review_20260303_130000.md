## 🔴 Critical Issues

### [C-1] AddToTsundokuUseCase に TOCTOU 競合がある

**ファイル**: `domain/usecase/AddToTsundokuUseCase.kt:24`

```kotlin
// ❌ 現在のコード
val currentCount = repository.observeTsundokuArticles().first().size
if (currentCount >= MAX_TSUNDOKU_SLOTS) return false

repository.updateStatus(articleId, ArticleStatus.TSUNDOKU)
```

**問題**: カウント確認（check）と書き込み（act）が別々の処理で、アトミックではない。
連打や複数タブ同時操作で「5件チェックを両方が通過 → 6件目が書き込まれる」が起こりえる。

**修正案**: DAO 側でアトミックに上限チェックを行う。

```kotlin
// ✅ 修正案A: DAO に上限付き UPDATE を追加する
// ArticleDao
@Query("""
    UPDATE articles SET status = :newStatus
    WHERE id = :id
    AND (SELECT COUNT(*) FROM articles WHERE status = 'TSUNDOKU') < :maxSlots
""")
suspend fun updateStatusIfSlotAvailable(id: String, newStatus: String, maxSlots: Int): Int

// UseCase
suspend operator fun invoke(articleId: String): Boolean {
    val updated = repository.updateStatusIfSlotAvailable(
        id = articleId,
        status = ArticleStatus.TSUNDOKU,
        maxSlots = MAX_TSUNDOKU_SLOTS,
    )
    return updated > 0
}
```

### [M-3] addToTsundoku の結果（Boolean）を ViewModel が無視している

**ファイル**: `ui/portal/PortalViewModel.kt:56`

```kotlin
// ❌ 現在のコード
fun addToTsundoku(articleId: String) {
    viewModelScope.launch {
        addToTsundokuUseCase(articleId)  // Boolean が返るが無視
    }
}
```

UseCase が `false`（スロット満杯）を返した場合や例外が発生した場合に UI へのフィードバックが一切ない。
ボタンの `enabled = !isSlotFull` で防いでいるが、競合時（[C-1]）やエラー時に無音で失敗する。

**修正案**: 結果を SnackBar などで通知する。

```kotlin
// ✅ イベントで通知
private val _event = MutableSharedFlow<PortalEvent>()
val event: SharedFlow<PortalEvent> = _event.asSharedFlow()

fun addToTsundoku(articleId: String) {
    viewModelScope.launch {
        val added = addToTsundokuUseCase(articleId)
        if (!added) _event.emit(PortalEvent.SlotFull)
    }
}

sealed interface PortalEvent {
    data object SlotFull : PortalEvent
}
```
